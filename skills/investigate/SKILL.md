# 기술 조사

`$ARGUMENTS`에 대해 체계적으로 조사하고 판단 재료를 제공한다.

## 조사 절차

1. **정체 파악** — 무엇인지, 누가 만들었는지, 어떤 문제를 해결하는지
2. **공식 정보 수집** — GitHub README, 공식 문서, 설치 방법
3. **실제 사용 후기** — 커뮤니티 리뷰, 이슈, 불만 사항
4. **비용/리스크** — 가격, 토큰 소비, 라이선스, 알려진 문제
5. **대안 비교** — 같은 목적의 다른 도구와 비교표 작성
6. **현재 환경과의 적합성** — 기존 스택과 충돌 여부, 도입 난이도

## 출력 형식

### [대상 이름] 조사 결과

**한 줄 요약:** ...

**장점:**
- ...

**단점 / 리스크:**
- ...

**비용:**
- ...

**대안 비교:**

| | 이것 | 대안 A | 대안 B |
|---|---|---|---|

**판단 재료:**
- 도입하면 좋은 경우: ...
- 도입하지 않는 게 나은 경우: ...

## 주의사항

- 판단을 내리지 않는다. 재료만 제공한다
- 확인되지 않은 정보는 "미확인"으로 표시한다
- 가능하면 수치와 출처를 포함한다
